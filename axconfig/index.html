Platform-specific constants and parameters for ArceOS.

Currently supported platforms can be found in the platforms directory of the ArceOS root.

Constants

  • ARCH
    Architecture identifier.
  • FAMILY
    Platform family.
  • KERNEL_BASE_PADDR
    Base physical address of the kernel image.
  • KERNEL_BASE_VADDR
    Base virtual address of the kernel image.
  • MAX_USER_HEAP_SIZE
    The size of the user heap.
  • MAX_USER_STACK_SIZE
    The size of the user stack.
  • MMIO_REGIONS
    MMIO regions with format (base_paddr, size).
  • PCI_BUS_END
    End PCI bus number.
  • PCI_ECAM_BASE
    Base physical address of the PCIe ECAM space (should read from ACPI ‘MCFG’ table).
  • PCI_RANGES
    PCI device memory ranges (not used on x86).
  • PHYS_MEMORY_BASE
    Base address of the whole physical memory.
  • PHYS_MEMORY_END
    End address of the whole physical memory.
  • PHYS_MEMORY_SIZE
    Size of the whole physical memory.
  • PHYS_VIRT_OFFSET
    Linear mapping offset, for quick conversions between physical and virtual addresses.
  • PLATFORM
    Platform identifier.
  • SIGNAL_TRAMPOLINE
    The base address of the signal trampoline.
  • SMP
    Number of CPUs
  • TASK_STACK_SIZE
    Stack size of each task.
  • TESTCASE_MEMORY_SIZE
    Testcase memory size.
  • TESTCASE_MEMORY_START
    Testcase memory start address.
  • TICKS_PER_SEC
    Number of timer ticks per second (Hz). A timer tick may contain several timer interrupts.
  • TIMER_FREQUENCY
    Timer interrupt frequencyin Hz.
  • USER_HEAP_BASE
    The base address of the user heap.
  • USER_MEMORY_START
    user memory
  • USER_STACK_TOP
    The base address of the user stack. And the stack bottom is user-stack-top + max-user-stack-size.
  • VIRTIO_MMIO_REGIONS
    VirtIO MMIO regions with format (base_paddr, size).
